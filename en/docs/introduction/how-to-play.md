# How to Play

## The core mechanic

Ironwake uses a d20-based resolution system. When a character attempts something with an uncertain outcome and meaningful consequences, the GM calls for a check:

```
d20 + modifier vs. Target Number (TN)
```

If the result meets or exceeds the TN, the attempt succeeds. Otherwise, it fails.

## Ability checks

Each of the six attributes has an associated modifier:

```
Modifier = (Attribute Score − 10) ÷ 2, rounded down
```

When the GM calls for an ability check, you roll a d20 and add the relevant attribute modifier. The GM sets the TN based on the difficulty of the task. Ability checks can also have auto-success or auto-fail, eg. for trivial or impossible tasks.

## Contested rolls

When two characters are directly opposing each other, both roll. The higher result, including any relevant modifiers, wins. Ties go to the character with the higher relevant attribute score; if still tied, the situation remains unchanged.

## Advantage and disadvantage

[Circumstances][conditions] may grant **advantage** or impose **disadvantage**. Advantage and disadvantage both stack.

## Fiction first

The GM calls for rolls only when outcomes are uncertain and consequential. If there is no meaningful chance of failure, or no interesting consequence for success, no roll is needed — the GM can narrate the outcome.
